## Prerequisites

- PHP 8.2+
- Composer
- Node.js 18+ and npm
- MySQL database
- A [Stripe](https://stripe.com) account (for subscription features)

## Setup

<Steps>
  <Step title="Clone the repository">
    Clone the DentalConnect repository and navigate into the project directory.

    ```bash
    git clone https://github.com/a3524110303-cpu/Proyecto_Sakari_DentalConnect.git
    cd Proyecto_Sakari_DentalConnect
    ```
  </Step>

  <Step title="Install PHP dependencies">
    Install all PHP packages via Composer. This also runs `package:discover` automatically.

    ```bash
    composer install
    ```
  </Step>

  <Step title="Install JavaScript dependencies">
    Install the Node.js packages used by Vite for frontend asset compilation.

    ```bash
    npm install
    ```
  </Step>

  <Step title="Configure your environment">
    Create a `.env` file in the project root. This repository does not include a `.env.example`, so create the file manually or use the template in [Environment variables](/configuration/environment).

    Update the following key variables:

    | Variable | Description |
    |---|---|
    | `APP_URL` | Your local URL, e.g. `http://localhost:8000` |
    | `DB_HOST` | MySQL host, e.g. `127.0.0.1` |
    | `DB_DATABASE` | Name of your local MySQL database |
    | `DB_USERNAME` | MySQL username |
    | `DB_PASSWORD` | MySQL password |
    | `STRIPE_KEY` | Your Stripe publishable key |
    | `STRIPE_SECRET` | Your Stripe secret key |
    | `STRIPE_WEBHOOK_SECRET` | Webhook signing secret from your Stripe dashboard |

    <Note>
      Stripe keys are only required if you want to test subscription checkout locally. The rest of the app — patients, appointments, odontogram — works without them.
    </Note>
  </Step>

  <Step title="Generate the application key">
    Generate a unique `APP_KEY` for encryption and session security.

    ```bash
    php artisan key:generate
    ```
  </Step>

  <Step title="Run database migrations">
    This creates all tables — clinics, users, patients, appointments, odontogram, SaaS plans, and more — and seeds the three default subscription plans (Basic, Premium, Ultra).

    ```bash
    php artisan migrate
    ```

    <Note>
      The migration also seeds the `saas_planes` table with the three default plans: **Basico** ($499/mo), **Premium** ($999/mo), and **Ultra** ($1,999/mo). You will need to update `stripe_price_id` in that table with your actual Stripe Price IDs before accepting real payments.
    </Note>
  </Step>

  <Step title="Build frontend assets">
    Compile and bundle frontend assets for the development environment.

    ```bash
    npm run build
    ```
  </Step>

  <Step title="Start the development server">
    The `dev` script in `composer.json` starts all required processes concurrently: the Laravel app server, queue worker, Pail log viewer, and Vite dev server.

    ```bash
    composer run dev
    ```

    This runs the following command:

    ```bash
    npx concurrently -c "#93c5fd,#c4b5fd,#fb7185,#fdba74" "php artisan serve" "php artisan queue:listen --tries=1 --timeout=0" "php artisan pail --timeout=0" "npm run dev" --names=server,queue,logs,vite --kill-others
    ```

    Alternatively, you can start the processes individually in separate terminals:

    <CodeGroup>
      ```bash App server
      php artisan serve
      ```

      ```bash Queue worker
      php artisan queue:listen --tries=1 --timeout=0
      ```

      ```bash Vite dev server
      npm run dev
      ```
    </CodeGroup>
  </Step>
</Steps>

## Access the app

Open [http://localhost:8000](http://localhost:8000) in your browser. You will see the public landing page with the three subscription plans.

| URL | Description |
|---|---|
| `http://localhost:8000` | Public landing page with plan pricing |
| `http://localhost:8000/register` | Create a new clinic account |
| `http://localhost:8000/login` | Sign in to an existing account |
| `http://localhost:8000/dashboard` | Clinic dashboard (requires login) |
| `http://localhost:8000/admin/panel` | Global admin panel (requires `administrador` role) |

## Roles and credentials

DentalConnect uses four roles, all defined on the `usuarios_sistema` table:

| Role | Access level |
|---|---|
| `administrador` | Full access to clinic settings, staff management, and the global SaaS admin panel at `/admin/panel` |
| `doctor` | Patient records, appointments, treatments, odontogram |
| `recepcionista` | Appointment management and patient list view |
| `paciente` | Patient-facing access |

<Note>
  There are no seeded default credentials. Register a new account at `/register` — the first user you create will be associated with a new clinic record. Assign the `administrador` role directly in the database to access the admin panel.
</Note>
